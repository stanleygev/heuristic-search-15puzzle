# Heuristic Search in the 15-Puzzle

**CS 57200 — Heuristic Problem Solving · Final Project · Stanley Gevers · Track B (Optimization)**

Implementation and experimental comparison of heuristic search algorithms for optimally and bounded-suboptimally solving random 15-puzzle instances. Five enhancements over a Manhattan-distance A\* baseline are implemented, integrated, and evaluated on a stratified 24-instance benchmark with paired statistical tests.

## Reproducing the benchmark end-to-end

The full benchmark exercises all 8 solver configurations across all 24 instances at a 10-second per-instance time limit. On a typical laptop:

```bash
python src/main.py --mode benchmark --n 24 --time-limit 10 --partition 5-5-5
python src/main.py --mode stats
```

This produces `results/results.json`, `results/summary.csv`, `results/figures/fig{1..6}.png`, and the Wilcoxon comparison tables.

If you only have a few minutes, you can run a subset:

```bash
python src/main.py --mode quick --n 10 --time-limit 5
```

If the full benchmark is interrupted, use the incremental runner — it's idempotent per-config and saves after every 3 instances:

```bash
python src/run_benchmark.py idastar_pdb 24 10
python src/run_benchmark.py astar_pdb 24 10
python src/run_benchmark.py weighted_125 24 10
python src/run_benchmark.py weighted_15 24 10
python src/run_benchmark.py weighted_20 24 10
python src/run_benchmark.py wida_15 24 10
python src/run_benchmark.py linearconflict 24 5
python src/run_benchmark.py manhattan 24 5
python src/generate_figures.py
python src/main.py --mode stats
```

## CLI reference

```
python main.py --mode {test|quick|benchmark|stats}
               [--n N]                           number of instances (default 100)
               [--partition {5-5-5|6-5-4|7-8}]   PDB partition (default 5-5-5)
               [--time-limit SECONDS]            per-instance limit (default 60)
               [--seed N]                        RNG seed (default 42)
               [--cache PATH]                    PDB cache dir (default data/pdb_cache)
               [--results PATH]                  output JSON (default results/results.json)
               [--figures PATH]                  output figures dir (default results/figures)
```

| Mode | What it does |
|---|---|
| `test` | Runs A\* + Manhattan and A\* + LC on 6 hand-picked solvable test cases. Fast. No PDB needed. |
| `quick` | Builds (or loads) the 5-5-5 PDB and benchmarks 4 solvers on `n` instances. |
| `benchmark` | Builds (or loads) the chosen PDB and benchmarks all 8 solvers on `n` instances. Generates figures. |
| `stats` | Reads `results/results.json` and prints the paired Wilcoxon table for runtime and nodes. |

## Algorithms implemented

All five enhancements are non-trivial and are statistically compared against the baseline:

| # | Algorithm | Optimal? | Memory | Notes |
|---|---|---|---|---|
| baseline | A\* + Manhattan | yes | open/closed list | Required reference point |
| 1 | A\* + Linear Conflict | yes | open/closed list | Adds 2 per conflicting row/column pair |
| 2 | A\* + 5-5-5 PDB | yes | open/closed list + tables | Three lookup tables, additive |
| 3 | IDA\* + 5-5-5 PDB | yes | O(d) stack + tables | ~200x less RAM than A\*+PDB |
| 4 | Weighted A\* (w in {1.25, 1.5, 2.0}) | <= w x optimal | open/closed list + tables | Bounded suboptimal |
| 5 | Weighted IDA\* (WIDA\*, w=1.5) | <= 1.5 x optimal | O(d) stack + tables | Linear memory + bounded suboptimal |

`pdb.py` also exports `PARTITION_6_5_4` and `PARTITION_7_8`. The 7-8 partition is hardware-intensive (total tables on the order of 4–6 GB and tens of minutes to build); we used 5-5-5 throughout because the available environment did not support the larger partitions, consistent with the Milestone 2 contingency plan.

## Headline results (24-instance stratified benchmark, 10s time limit)

| Configuration | Solved | Mean nodes | Mean time | Mean ratio to optimal | Peak MB |
|---|---:|---:|---:|---:|---:|
| A\*+Manhattan | 4/24 | 4,454 | 0.599 s | 1.00 | 2.09 |
| A\*+LinearConflict | 4/24 | 1,558 | 1.285 s | 1.00 | 0.74 |
| A\*+PDB(5-5-5) | 23/24 | 7,878 | 0.815 s | 1.00 | 4.06 |
| IDA\*+PDB(5-5-5) | 24/24 | 19,219 | 0.910 s | 1.00 | **0.020** |
| WeightedA\*(w=1.25) | 24/24 | 1,569 | 0.159 s | 1.027 | 0.61 |
| WeightedA\*(w=1.5) | 24/24 | 748 | **0.079 s** | 1.096 | 0.26 |
| WeightedA\*(w=2.0) | 24/24 | 715 | 0.074 s | 1.209 | 0.20 |
| WIDA\*(w=1.5) | 24/24 | 1,748 | 0.127 s | 1.165 | **0.020** |

On the 23-instance subset solved by both A\*+PDB and the weighted variants, paired Wilcoxon signed-rank tests with Bonferroni correction find every A\*+PDB-vs-faster-variant comparison significant at p < 0.05; differences among the suboptimal variants themselves are not.

## Reproducibility

- Benchmark seed: `42` (stored in `data/benchmark.json`)
- Stratified by Manhattan-distance tier: 8 easy / 8 medium / 8 hard
- All randomness goes through `random.Random(seed)`; no global RNG used
- PDB tables are deterministic given the partition; cached to disk after first build
- Fixed time and memory limits; all measurements run on a single machine
- All figures and tables in the report regenerate from `results/results.json`
